# M1 — Lexer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Port `gdproto/lexer.py` to Go in `internal/lexer`, with TDD red/green for every behavior. End state: `internal/lexer.Tokenize(source, filename string) ([]Token, error)` produces the same token stream and the same error positions as the Python lexer for every fixture in `~/foss/gdproto/tests/test_lexer.py`.

**Architecture:** A `Lexer` struct holds source, position, line, column, and accumulated tokens. Behavior is split across small unexported methods (cursor, scanners). The single exported function `Tokenize` constructs a Lexer and runs it. Tokens are plain values; errors are a `*LexerError` type implementing `error` with `File/Line/Column/Message`. No goroutines, no I/O, no global state.

**Tech Stack:** Standard library only. `unicode` for character classes (`unicode.IsLetter`, `unicode.IsDigit`); the proto3 grammar is ASCII-only for identifiers but the Python implementation uses `str.isalpha`/`str.isdigit`/`str.isalnum`, which are Unicode-aware. We deliberately match by using Go's `unicode` package — but ASCII is sufficient for everything proto3 cares about. Use `unicode.IsLetter`/`IsDigit` so we don't silently diverge on edge cases.

**Reference:**
- Python lexer: `~/foss/gdproto/src/gdproto/lexer.py` (548 lines)
- Python tests: `~/foss/gdproto/tests/test_lexer.py` (~30 tests, 8 test classes)
- Design doc: [docs/superpowers/specs/2026-04-27-gogdproto-design.md](../specs/2026-04-27-gogdproto-design.md)

## File Structure

| Path                                | Responsibility                                                |
|-------------------------------------|---------------------------------------------------------------|
| `internal/lexer/doc.go`             | Existing package comment (don't modify).                      |
| `internal/lexer/token.go`           | `TokenType` enum, `Token` struct, keyword table, `String()`.  |
| `internal/lexer/error.go`           | `LexerError` type with `Error()` method.                      |
| `internal/lexer/lexer.go`           | `Lexer` struct, cursor methods, `Tokenize` driver.            |
| `internal/lexer/lexer_test.go`      | Unit tests (mirror Python `tests/test_lexer.py`).             |

We split into 4 files (excluding doc.go) so each has one clear responsibility. `lexer.go` will end up ~300 lines of Go; that's acceptable for a state machine where splitting further would scatter cursor state across files.

The exported API is exactly:

```go
package lexer

type TokenType int

const (
    TokenEOF TokenType = iota
    TokenSyntax
    // ... etc
)

type Token struct {
    Type   TokenType
    Value  string
    Line   int
    Column int
}

type LexerError struct {
    File    string
    Line    int
    Column  int
    Message string
}

func (e *LexerError) Error() string

func Tokenize(source, filename string) ([]Token, error)
```

`Tokenize` returns `([]Token, *LexerError)` packaged as `error`. Callers test with `errors.As`.

**Naming convention:** Token types use the `Token<Name>` prefix (e.g. `TokenSyntax`, `TokenIntLiteral`, `TokenLBrace`). This avoids the Python-style `TokenType.SYNTAX` and stays idiomatic Go.

package lexer

import "fmt"

// LexerError describes a tokenization failure.
type LexerError struct {
	File    string
	Line    int
	Column  int
	Message string
}

// Error formats the error as "<file>:<line>:<col>: error: <message>".
// If File is empty, "<input>" is used.
func (e *LexerError) Error() string {
	file := e.File
	if file == "" {
		file = "<input>"
	}
	return fmt.Sprintf("%s:%d:%d: error: %s", file, e.Line, e.Column, e.Message)
}

package lexer

import "fmt"

// Tokenize converts .proto source code into a stream of tokens.
// The filename is used only in error messages; pass "" for "<input>".
// The returned slice always ends with a TokenEOF entry.
func Tokenize(source, filename string) ([]Token, error) {
	l := &lexer{source: source, filename: filename, line: 1, column: 1}
	return l.run()
}

type lexer struct {
	source   string
	filename string
	pos      int
	line     int
	column   int
	tokens   []Token
}

// run drives tokenization. Each iteration consumes whitespace then dispatches
// on the next significant character.
func (l *lexer) run() ([]Token, error) {
	for l.pos < len(l.source) {
		l.skipWhitespace()
		if l.pos >= len(l.source) {
			break
		}

		ch := l.source[l.pos]
		line, col := l.line, l.column

		if t, ok := singleCharSymbol(ch); ok {
			l.tokens = append(l.tokens, Token{Type: t, Value: string(ch), Line: line, Column: col})
			l.advance()
			continue
		}

		return nil, &LexerError{
			File:    l.filename,
			Line:    line,
			Column:  col,
			Message: fmt.Sprintf("Unexpected character: %q", ch),
		}
	}

	l.tokens = append(l.tokens, Token{Type: TokenEOF, Line: l.line, Column: l.column})
	return l.tokens, nil
}

// advance consumes the current byte, updating line/column.
func (l *lexer) advance() {
	if l.pos >= len(l.source) {
		return
	}
	ch := l.source[l.pos]
	l.pos++
	if ch == '\n' {
		l.line++
		l.column = 1
	} else {
		l.column++
	}
}

// skipWhitespace consumes spaces, tabs, newlines, and CR.
func (l *lexer) skipWhitespace() {
	for l.pos < len(l.source) {
		ch := l.source[l.pos]
		if ch == ' ' || ch == '\t' || ch == '\n' || ch == '\r' {
			l.advance()
			continue
		}
		return
	}
}

func singleCharSymbol(ch byte) (TokenType, bool) {
	switch ch {
	case '{':
		return TokenLBrace, true
	case '}':
		return TokenRBrace, true
	case '[':
		return TokenLBracket, true
	case ']':
		return TokenRBracket, true
	case '(':
		return TokenLParen, true
	case ')':
		return TokenRParen, true
	case '<':
		return TokenLT, true
	case '>':
		return TokenGT, true
	case ';':
		return TokenSemicolon, true
	case '=':
		return TokenEquals, true
	case ',':
		return TokenComma, true
	case '.':
		return TokenDot, true
	}
	return 0, false
}
```

Note: we intentionally treat `source` as a byte slice for ASCII-only proto3. We'll switch to runes only for identifier scanning if Task 2 needs it (it doesn't — proto3 identifiers are ASCII).

func isDigit(ch byte) bool { return ch >= '0' && ch <= '9' }

func isHexDigit(ch byte) bool {
	return isDigit(ch) || (ch >= 'a' && ch <= 'f') || (ch >= 'A' && ch <= 'F')
}

// peek returns the byte at l.pos+offset, or 0 if out of bounds.
func (l *lexer) peek(offset int) byte {
	pos := l.pos + offset
	if pos < 0 || pos >= len(l.source) {
		return 0
	}
	return l.source[pos]
}

func (l *lexer) readNumber() Token {
	startLine, startCol := l.line, l.column
	start := l.pos

	if l.source[l.pos] == '-' {
		l.advance()
	}

	// Hex: 0x...
	if l.source[l.pos] == '0' && l.pos+1 < len(l.source) && (l.source[l.pos+1] == 'x' || l.source[l.pos+1] == 'X') {
		l.advance() // 0
		l.advance() // x
		for l.pos < len(l.source) && isHexDigit(l.source[l.pos]) {
			l.advance()
		}
		return Token{Type: TokenIntLiteral, Value: l.source[start:l.pos], Line: startLine, Column: startCol}
	}

	// Octal: leading 0 followed by another digit
	if l.source[l.pos] == '0' && l.pos+1 < len(l.source) && isDigit(l.source[l.pos+1]) {
		l.advance() // first 0
		for l.pos < len(l.source) {
			ch := l.source[l.pos]
			if ch < '0' || ch > '7' {
				break
			}
			l.advance()
		}
		return Token{Type: TokenIntLiteral, Value: l.source[start:l.pos], Line: startLine, Column: startCol}
	}

	// Decimal integer part
	for l.pos < len(l.source) && isDigit(l.source[l.pos]) {
		l.advance()
	}

	isFloat := false
	if l.pos < len(l.source) && l.source[l.pos] == '.' {
		isFloat = true
		l.advance()
		for l.pos < len(l.source) && isDigit(l.source[l.pos]) {
			l.advance()
		}
	}

	// Exponent
	if l.pos < len(l.source) && (l.source[l.pos] == 'e' || l.source[l.pos] == 'E') {
		isFloat = true
		l.advance()
		if l.pos < len(l.source) && (l.source[l.pos] == '+' || l.source[l.pos] == '-') {
			l.advance()
		}
		for l.pos < len(l.source) && isDigit(l.source[l.pos]) {
			l.advance()
		}
	}

	tt := TokenIntLiteral
	if isFloat {
		tt = TokenFloatLiteral
	}
	return Token{Type: tt, Value: l.source[start:l.pos], Line: startLine, Column: startCol}
}

func (l *lexer) readString() (Token, error) {
	startLine, startCol := l.line, l.column
	quote := l.source[l.pos]
	l.advance() // opening quote

	var sb strings.Builder
	for {
		if l.pos >= len(l.source) {
			return Token{}, &LexerError{
				File:    l.filename,
				Line:    startLine,
				Column:  startCol,
				Message: "Unterminated string literal",
			}
		}

		ch := l.source[l.pos]

		if ch == quote {
			l.advance() // closing quote
			return Token{Type: TokenStringLiteral, Value: sb.String(), Line: startLine, Column: startCol}, nil
		}

		if ch == '\n' {
			return Token{}, &LexerError{
				File:    l.filename,
				Line:    l.line,
				Column:  l.column,
				Message: "Newline in string literal",
			}
		}

		if ch == '\\' {
			l.advance()
			if l.pos >= len(l.source) {
				return Token{}, &LexerError{
					File:    l.filename,
					Line:    l.line,
					Column:  l.column,
					Message: "Unterminated escape sequence",
				}
			}
			escaped := l.source[l.pos]
			switch escaped {
			case 'n':
				sb.WriteByte('\n')
				l.advance()
			case 'r':
				sb.WriteByte('\r')
				l.advance()
			case 't':
				sb.WriteByte('\t')
				l.advance()
			case '\\':
				sb.WriteByte('\\')
				l.advance()
			case '\'':
				sb.WriteByte('\'')
				l.advance()
			case '"':
				sb.WriteByte('"')
				l.advance()
			case '0':
				sb.WriteByte(0)
				l.advance()
			case 'x':
				l.advance() // x
				if l.pos+1 >= len(l.source) || !isHexDigit(l.source[l.pos]) || !isHexDigit(l.source[l.pos+1]) {
					return Token{}, &LexerError{
						File:    l.filename,
						Line:    l.line,
						Column:  l.column,
						Message: "Invalid hex escape sequence",
					}
				}
				v, err := strconv.ParseUint(l.source[l.pos:l.pos+2], 16, 8)
				if err != nil {
					return Token{}, &LexerError{
						File:    l.filename,
						Line:    l.line,
						Column:  l.column,
						Message: "Invalid hex escape sequence",
					}
				}
				sb.WriteByte(byte(v))
				l.advance()
				l.advance()
			default:
				return Token{}, &LexerError{
					File:    l.filename,
					Line:    l.line,
					Column:  l.column,
					Message: fmt.Sprintf("Invalid escape sequence: \\%c", escaped),
				}
			}
			continue
		}

		sb.WriteByte(ch)
		l.advance()
	}
}
```

func TestMultilineColumnTracking(t *testing.T) {

	cases := []struct{ line, col int }{
		{1, 1}, {2, 3}, {3, 5},
	}
	for i, c := range cases {
		if tokens[i].Line != c.line || tokens[i].Column != c.col {

		}
	}
}

**Risks worth flagging:**

1. **Negative-number ambiguity.** Python treats `-` as part of a number only if the next char is a digit (no whitespace allowed between). We match that. But the *parser* will need to handle `field = -5;` and `field = - 5;` differently — the first is an int literal, the second is two tokens (`-`, `5`). Python errors on bare `-` because it has no symbol-table entry. We'll match that behavior; if M2 (parser) needs `-` as a token, it'll need a lexer change. Note in M2 plan when written.
2. **`\0` escape.** Python writes `\0` as the NUL byte `chr(0)`. Our `sb.WriteByte(0)` matches; downstream code must tolerate NUL bytes in `Token.Value`.
3. **Octal vs leading-zero decimal.** `0` alone is decimal `IntLiteral` "0". `00` would be Python-octal "00". Currently our code matches (peek check on the second char). Edge case: `0.5` → not octal because the second char is `.`, not a digit → decimal float. Verified by `TestFloatDecimal`-adjacent tests.
4. **Coverage budget.** We don't test every `TokenType` constant individually for `String()`. If revive's `exhaustive` hits, we'll add a coverage-only test. Not in plan because it's noise; add only if lint/coverage flags it.
