Naming: Go exports use TitleCase. `to_gdscript` → `ToGDScript`. `is_const` → `IsConst`. `type_hint` → `TypeHint`. Helper functions match Python: `Lit`, `V`, `Var`, etc. (capitalized). Use `_` suffix for `And`, `Or`, `Not` since Python uses `and_`, `or_`, `not_` to avoid keyword collision; in Go those aren't keywords, but for parity we use `And`, `Or`, `Not` (no underscore).

The `Node` interface:

```go
package gdast

type Node interface {
    ToGDScript(indentLevel int) string
}

type Expression interface { Node; expression() }
type Statement interface { Node; statement() }
```

The `expression()` and `statement()` methods are unexported markers (a Go idiom for sealed interfaces). Each concrete type implements its appropriate marker(s).

```go
// node.go
package gdast

import "strings"

type Node interface { ToGDScript(indentLevel int) string }
type Expression interface { Node; expression() }
type Statement interface { Node; statement() }

func indent(level int) string { return strings.Repeat("\t", level) }
```

```go
// In functions.go (just GDScript root for now):
type GDScript struct { Items []Node }
func (s GDScript) ToGDScript(indentLevel int) string {
    var sb strings.Builder
    for i, item := range s.Items {
        if i > 0 { sb.WriteByte('\n') }
        sb.WriteString(item.ToGDScript(indentLevel))
    }
    return sb.String()
}
```

```go
// In statements.go (no-arg statements):
type EmptyLine struct{}
func (EmptyLine) ToGDScript(int) string { return "" }
func (EmptyLine) statement() {}

type PassStatement struct{}
func (PassStatement) ToGDScript(level int) string { return indent(level) + "pass" }
func (PassStatement) statement() {}

type BreakStatement struct{}
func (BreakStatement) ToGDScript(level int) string { return indent(level) + "break" }
func (BreakStatement) statement() {}

type ContinueStatement struct{}
func (ContinueStatement) ToGDScript(level int) string { return indent(level) + "continue" }
func (ContinueStatement) statement() {}

// RawStatement: prepend indent to each non-empty line (matches Python).
type RawStatement struct{ Code string }
func (r RawStatement) ToGDScript(level int) string {
    if r.Code == "" { return "" }
    if level == 0 { return r.Code }
    lines := strings.Split(r.Code, "\n")
    out := make([]string, len(lines))
    pad := indent(level)
    for i, line := range lines {
        if strings.TrimSpace(line) != "" { out[i] = pad + line } else { out[i] = line }
    }
    return strings.Join(out, "\n")
}
func (RawStatement) statement() {}
```

```go
// In expressions.go (RawExpression only for now):
type RawExpression struct{ Code string }
func (r RawExpression) ToGDScript(level int) string {
    if level == 0 { return r.Code }
    if !strings.Contains(r.Code, "\n") { return indent(level) + r.Code }
    lines := strings.Split(r.Code, "\n")
    out := make([]string, len(lines))
    pad := indent(level)
    for i, line := range lines {
        if strings.TrimSpace(line) != "" { out[i] = pad + line } else { out[i] = line }
    }
    return strings.Join(out, "\n")
}
func (RawExpression) expression() {}
```

```go
func TestLiteralInt(t *testing.T) {
    if got := (Literal{Value: 42}).ToGDScript(0); got != "42" { t.Errorf("got %q", got) }
}
func TestLiteralBoolTrue(t *testing.T) {
    if got := (Literal{Value: true}).ToGDScript(0); got != "true" { t.Errorf("got %q", got) }
}
func TestLiteralNil(t *testing.T) {
    if got := (Literal{Value: nil}).ToGDScript(0); got != "null" { t.Errorf("got %q", got) }
}
func TestLiteralStringDoubleQuoted(t *testing.T) {
    if got := (Literal{Value: "hello"}).ToGDScript(0); got != `"hello"` { t.Errorf("got %q", got) }
}
func TestLiteralStringWithDoubleQuote(t *testing.T) {
    // Has " but no ' → use '...'
    if got := (Literal{Value: `say "hi"`}).ToGDScript(0); got != `'say "hi"'` { t.Errorf("got %q", got) }
}
func TestLiteralStringWithBoth(t *testing.T) {
    // Has both quotes → escape and use "..."
    if got := (Literal{Value: `it's "hot"`}).ToGDScript(0); got != `"it's \"hot\""` { t.Errorf("got %q", got) }
}
func TestVariable(t *testing.T) { ... }
func TestBinaryOp(t *testing.T) { ... }
// ... etc
```

**Implementation guidance:**

`Literal.Value` is `any`. Use a type switch:
- `nil` → `"null"`
- `bool` → `"true"`/`"false"`
- `string` → smart-quote logic from Python:
  - `"` in value AND `'` not in value → `'value'`
  - both quotes present → escape `\` and `"`, use `"value"`
  - else → `"value"`
- `float64` (and `float32`):
  - `math.IsInf(v, 1)` → `"INF"`
  - `math.IsInf(v, -1)` → `"-INF"`
  - `math.IsNaN(v)` → `"NAN"`
  - else `strconv.FormatFloat(v, 'g', -1, 64)` BUT match Python's `str(float)` — Python's `str(3.14)` returns `"3.14"`. Use `strconv.FormatFloat(v, 'g', -1, 64)` and verify by experiment. **For perfect parity, you may need to use Python-compatible float formatting; if the simple form fails, fall back to `strconv.FormatFloat(v, 'g', -1, 64)` and document any divergence.**
- `int`, `int32`, `int64`, `uint*` → `fmt.Sprintf("%d", v)`
- default → `fmt.Sprint(v)`

`Call.Function` is `any` (string or `Expression`); type-switch.

For `BinaryOp`/`UnaryOp`/`TypeCast`/`TernaryOp`/`GetAttr`/`Subscript`: just format string concatenations of nested `ToGDScript(0)` calls.

`Array` and `Dictionary`: format inline (no multi-line). Empty → `[]` and `{}`.

## Task 5: Helper constructors

**Goal:** Port `helpers.py` — convenience constructors for common patterns.

**Files:** `helpers.go`, `gdast_test.go`.

**Reference Python:** `helpers.py:21-186`.

**Acceptance:** ~10 tests verifying each helper creates the right node:

```go
func TestLitInt(t *testing.T) {
    n := Lit(42)
    if l, ok := n.(Literal); !ok || l.Value != 42 {
        t.Errorf("got %T %+v", n, n)
    }
}
func TestVHelper(t *testing.T) {
    n := V("foo")
    if v, ok := n.(Variable); !ok || v.Name != "foo" {
        t.Errorf("got %T %+v", n, n)
    }
}
// ... and so on for Call, Attr, Var, Const, Assign, Ret, If, While, For, Eq, Ne, Lt, Gt, Le, Ge, And, Or, Not, Add, Sub, Mul, Div.
```

**Implementation guidance:**

```go
package gdast

func Lit(value any) Literal { return Literal{Value: value} }
func V(name string) Variable { return Variable{Name: name} }
func Call(function any, args ...Expression) CallExpression { return CallExpression{Function: function, Arguments: args} }
// (Note: `Call` is a type name AND helper. We can rename the type to `CallExpression` and keep the helper as `Call`.
//  Alternative: name type `Call` and helper as `CallExpr`. Pick one. The plan suggests `Call` type, helper called as `MakeCall`.
//  Simpler: rename type to `CallExpr` and keep helper `Call`.)
```

**Important naming decision:** Python has `Call` as both class and function — Python allows this. Go does not. **Pick one approach:**
- **Option A:** Name the type `Call` and the helper... won't compile (collision).
- **Option B:** Type `CallExpr`, helper `Call`. — RECOMMENDED.
- **Option C:** Type `Call`, no helper (use struct literal directly).

**Use Option B.** Same applies to `If`, `While`, `For`, `Var`, `Const`, `Assign`, `Ret`, `And`, `Or`, `Not`. Suggested naming:
- Type `IfStatement` → helper `If`
- Type `WhileStatement` → helper `While`
- Type `ForStatement` → helper `For`
- Type `VarDeclaration` → helper `Var` (and `Const`)
- Type `Assignment` → helper `Assign`
- Type `ReturnStatement` → helper `Ret`
- Type `Call` would conflict — RENAME TYPE TO `CallExpr`. Helper remains `Call`.
- `Eq`, `Ne`, `Lt`, `Gt`, `Le`, `Ge`, `And`, `Or`, `Not`, `Add`, `Sub`, `Mul`, `Div` are helpers — no type conflicts.
- `Attr` helper → wraps `GetAttr` type.

**Steps:** standard TDD. Commit: `feat(gdast): helper constructors`.

## Self-Review

**Spec coverage:** Every Python `nodes.py` and `helpers.py` symbol is mapped to a Go equivalent in T0–T5. T6 verifies end-to-end.

**Risks:**

1. **Float formatting parity:** Python's `str(3.14)` and Go's `strconv.FormatFloat(3.14, 'g', -1, 64)` may differ for some values. T1 should add a test comparing expected outputs and adjust if needed.
2. **Empty-statement body emit "pass":** Python's `WhileStatement` and `ForStatement` and `MatchCase` emit `pass` for empty bodies. Don't forget this.
3. **`Comment` block stripping:** Python strips leading/trailing whitespace from the entire block before splitting. Match exactly.
4. **`ClassDefinition` blank-line interleaving:** Python adds blank lines between members but not after the last. Critical for diff parity in T6.
5. **Helper naming collisions:** documented in T5. Use `CallExpr` type / `Call` helper.
